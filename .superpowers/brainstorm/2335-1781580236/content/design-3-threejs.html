3D 力导向布局策略

节点位置计算：混合力导向 + 约束锚定

不用 3d-force-graph，但借鉴其力导向算法思想，自己写简化版：

力导向规则（每帧 tick）：
  ● 排斥力：所有节点互斥 → 防止重叠
  ● 弹簧力：有连线的节点互相吸引 → 保持拓扑关系
  ● 锚定力：master 线节点沿 X 轴分布，锚定在 x = eventIndex * 1.5
  ● 分支力：分支线节点从分叉点向 Z- 轴偏移，y 轴轻微下弯
  ● 阻尼：速度 × 0.95，防止抖动

初始化布局：
  master[0] = (0,    0, 0)    ← 2010 分叉点
  master[1] = (1.5,  0, 0)    ← 2010 签约
  master[2] = (3.0,  0, 0)    ← 2011 总决赛
  ...
  branch[0] = (0,    0, 0)    ← 共享分叉点
  branch[1] = (0.3, -0.5, -1.2)  ← 从分叉点向下 + 向后
  branch[2] = (1.0, -0.8, -2.0)
  ...

粒子消散动画（Dissolve）

当用户选择非真实选项 → master 线节点逐级消散

流程：

  1. 克隆要被消散的节点/连线的 geometry → 转成点数据
  2. 创建临时 BufferGeometry（粒子云，每节点 200-500 粒子）
  3. ShaderMaterial：opacity 从 1→0, position 从原始位置 → 随机散射方向
  4. GSAP timeline：每个节点间隔 150ms 开始消散，时长 800ms
  5. 消散完成后移除原 mesh + 粒子 group，dispose geometry
// 简化 Shader
vertexShader: `
  attribute float aRandomAngle;
  uniform float uProgress;    // 0→1
  uniform vec3 uOrigin;
  varying float vAlpha;
  void main() {
    vec3 scatter = vec3(
      sin(aRandomAngle * 6.28) * uProgress * 3.0,
      cos(aRandomAngle * 6.28) * uProgress * 2.0,
      (aRandomAngle - 0.5) * uProgress * 2.0
    );
    vec3 pos = uOrigin + scatter;
    vAlpha = 1.0 - uProgress;
    gl_Position = projectionMatrix * modelViewMatrix * vec4(pos, 1.0);
    gl_PointSize = 4.0 * (1.0 - uProgress * 0.5);
  }
`

相机系统

OrbitControls + 自动飞入
默认视角：俯视 30°，距离 8 单位
OrbitControls：
  - 鼠标滚轮：缩放 (2～20 单位)
  - 左键拖拽：旋转（限制 360° 水平 / 15°～75° 俯仰）
  - 右键拖拽：平移
  - enableDamping: true (惯性)

自动飞入（GSAP）：
  - 点击节点 → camera 飞到节点前方 2 单位
  - 双击空白 → camera 回到默认视角
  - 分支切换 → camera 飞向新分支的第一个节点

视锥剔除：
  - 距离 > 15 单位的节点不渲染
  - 屏幕外节点 skip 粒子计算

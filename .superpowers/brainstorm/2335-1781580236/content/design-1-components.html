设计 §1：组件树 & 项目结构

Zustand 单一 store → Three.js 引擎 + React 浮层

数据流路径

SSE Stream (FastAPI)
    │  event_type: "timeline_event" | "stat_update" | "social_post" | "[DONE]"
    │
    ▼
useSSE() hook
    │  解析 SSE → 按 event_type 分发到 store 对应切片
    │
    ▼
┌──────────── ZUSTAND STORE ────────────┐
│                                        │
│  graphSlice          rpgSlice          │
│  - nodes[]           - championships   │
│  - edges[]           - legacy           │
│  - activeBranch      - mediaFavor       │
│  - masterOpacity     - fanReputation    │
│  - choiceId          - capHealth        │
│                      - physicalToll     │
│  socialSlice         uiSlice            │
│  - posts[]           - phase            │
│  - isStreaming       - selectedChoice   │
│  - feedVisible       - error            │
│                                        │
└───┬──────────┬──────────┬──────────────┘
    │          │          │
    ▼          ▼          ▼
┌───────┐ ┌───────┐ ┌─────────┐
│ThreeJS│ │Hexagram│ │Social   │
│Canvas │ │Radar   │ │Feed     │
│(r3f)* │ │(SVG)   │ │(DOM)    │
└───────┘ └───────┘ └─────────┘

* 纯 Three.js，不用 r3f。保持完全控制。

组件层级

App.tsx
├── ThreeCanvas.tsx          ← 占据全屏，Three.js 渲染目标
│   ├── StarField.ts         ← 星空粒子背景（ShaderMaterial）
│   ├── DestinyGraph.ts      ← 力导向图：节点 + 连线
│   │   ├── ForkNode.ts      ← 分叉点球体 + 光环脉动
│   │   ├── EventNode.ts     ← 事件小球体
│   │   └── EdgeLine.ts      ← 连线 + 粒子流动光效
│   ├── ParticleSystem.ts    ← 消散/聚合粒子（BufferGeometry）
│   └── CameraController.ts  ← 轨道控制 + 飞入动画
│
├── ChoiceWheel.tsx          ← HTML overlay，分叉点弹出
│   └── ChoiceCard.tsx       ← 单个选项卡片
│
├── HexagramRadar.tsx        ← 右下角 SVG 六芒星
│
├── SocialFeed.tsx           ← 左侧 HTML 瀑布流
│   └── SocialPost.tsx       ← 单条虚拟推文
│
└── StatusBar.tsx            ← 底部进度文案

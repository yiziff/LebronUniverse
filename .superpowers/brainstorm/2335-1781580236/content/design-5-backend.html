新增 §1：RPG 状态变动

对于你生成的每个事件，评估它对勒布朗·詹姆斯六维状态的影响：
- championships: 总冠军数 (0-7)
- legacy: 历史地位 (0-100)
- media_favor: 媒体好感度 (0-100)
- fan_reputation: 球迷口碑 (0-100)
- cap_health: 球队薪资健康度 (0-100，越高越健康)
- physical_toll: 体能透支率 (0-100，越高越透支)

变动规则：
- 加盟大市场球队 → media_favor +20~30, fan_reputation +10~15
- 组建超级球队 → cap_health -15~25, legacy +5~10
- 季后赛走远 → physical_toll +10~20
- 赢得总冠军 → championships +1, legacy +15~25
- 失败/横扫 → media_favor -10~20, fan_reputation -5~10

新增 §2：虚拟社交舆论

在推演完所有事件后，生成 3-5 条模拟的社交媒体反应：

角色配比：
- 1 条来自 NBA 名嘴（Stephen A. Smith / Skip Bayless 风格）
- 1 条来自涉及的球队死忠球迷（极度亢奋或极度愤怒）
- 1 条来自"詹黑"阵营 — 用经典的勒布朗黑梗：
  "Who's your daddy?"、"LeBronto"、"Buster"、"🐐❌"、"猩猩"、
  "没有大腿不会打球"、"决定1/2/3/N"、"not 1, not 2..."
- 0-1 条来自中立媒体 / 记者
- 0-1 条恶搞风格

风格要求：
- 还原真实 NBA 推特的火药味和幽默感
- 中文 + 英文混用（"LEBRON TO THE KNICKS?! 纽约球迷过年了！！"）
- 每条 ≤ 140 字（模拟推特长度）
- 情感标签: angry / excited / sarcastic / shocked / hate

# 白名单：从 players.json + 手动扩展
VALID_PLAYERS = {
    "lebron-james", "dwyane-wade", "chris-bosh", "derrick-rose",
    "amare-stoudemire", "carmelo-anthony", "chris-paul", "dwight-howard",
    "dirk-nowitzki", "kevin-durant", "kobe-bryant", "paul-george",
    "john-wall", "demarcus-cousins", "gordon-hayward",
    # 2010 年联盟其他关键球员（扩展至 ~80 人）
    "kyrie-irving", "klay-thompson", "draymond-green",
    "kawhi-leonard", "damian-lillard", "anthony-davis",
    ...
}

def validate_events(events: list[dict]) -> tuple[list[dict], list[str]]:
    """
    校验 AI 生成的每个事件
    返回: (通过的事件, 被拒绝的实体列表)
    """
    rejected = []
    clean_events = []

    for event in events:
        # 提取事件中所有人名
        names = extract_player_names(event)
        unknowns = [n for n in names if n not in VALID_PLAYERS]

        if unknowns:
            rejected.extend(unknowns)
            # 整个事件被标记为 rejected
            # → 触发 LLM 重试，附带错误信息
        else:
            clean_events.append(event)

    return clean_events, rejected

错误恢复策略

3 层容错
层级场景行为
L1 LLM 发明不存在球员 validator 拦截 → 重试（Prompt 注入禁用名单）→ 最多 3 次
L2 LLM 返回空/格式错 fallback 解析 → 重试 → 3 次后返回已有部分事件 + warning
L3 API 挂了 / 超时 SSE error 事件 → 前端显示"AI 暂时不可用" + 重试按钮
